Odoo your NTTY

Integrate with NTTY, where your products live

Let's you easily import your entities from NTTY and transform them into products in Odoo. Based on settings realted to the module, you can control how and what you should create.

NTTY Import with ease

Use the NTTY Import wizard to import your entities from NTTY and transform them into Products in Odoo. The wizard uses the entity identifier to gather the entity information, such as, name, product owner, values etc. Depending on your settings and the entity itself, a supplier list will be rendered and one product will be created pr supplier selected. The product owner can also be tied to your product brand.

Keep your products up to date

By invoking the Product Lifcycle Management module, you will be able to track the status of your product. It will be synced with NTTY's own lifecycle by using a background job included in this module. However if you desire to control the update manually, we have also included a simple step to do so.

Configuring background job

The module have 4 stages and each stage of the lifecycle from NTTY, is mapped to one of these stages through the Lifecycle Mappings.

Update lifecycle manually

Simply navigate to sales/NTTY/update Lifecycle. From there it is pretty obvious, but just to be safe, press the freakin' button

Import customers/companies

If desired, you can import your customers based on how you have defined your entity in NTTY. This will also run as a bacground job, based on your settings.

The list of customers/companies are gathered based on the article's organization and the organization of each part number tied to the article.

NTTY Configuration

Since entity retrieval will be done thorugh API calls to NTTY, we need configure the connection. You will be required to enter:

  • NTTY service address: http://www.ntty.com/api/v1/
  • NTTY service user email: email address of the NTTY user
  • NTTY service token: the authorization token of the NTTY user

    • Product settings

    Check Suppliers

    Will give you the possibility to import and create products one a ome-product-per-supplier level, provided the Supplier matching in NTTY has been run or the values with the key supplier_matching has been created.

    Generate Pricelist

    Will enable the feature to generate pricelists

    Include Related Products

    If this is set to True, the Import will chech and create a set of related products under your Sales on your Product in Odoo. The products related are based on the entity identifier. There is a possibility enter related products manually as well.

    Include Partner Information

    Will include any partner information available from NTTY.

    Include Product Category

    Will include the product categories defined in NTTY as Certifications/Requirements named Thecknology (Parent) and Category (Child). If the Technology/Category does not exist in Odoo, we will create it while creating the product. The categoriztion is post fixed to Odoo categories defined above, and if there is no category in NTTY, the category for that product will be set to the one defined in Odoo. In axample one could have predefined: All/Sports and the product from Entity could be Clothing/Shorts. The Product Category three will then look like this: All/Sports/Clothing/Shorts

    Include Supplier Shortnane - Revise title

    If this one is set to to true, we will use the Shortname, if it is defined, or the Supplier name to suffix the product name. A great festure for recognizing diff products by their supplier.

    Product variant settings - Manual step

    Each entity can contain a set of values. The values are Key-Value pairs and can be brought over to your product in Odoo by adding the key to your product attribute. For each match we have, the value of this Key will be created under the Product Variants tab on the Product itself. The key-value pair for Supplier Matching is by default used for processing the supplier list that you can create a produt from. If one wants to maintain a full list of all Suppliers in the supplier matching, it needs to be created as an attribute.

  • Determine your Key-Value pairs
  • Create Attributes
  • View values

    • Procurement/WMS

    We use these settings to determine the procurement method to initially assign your product. These can be overwritten on a product by product basis. If you wish to disable this, there will simply be nothing set for Procurement and Routes. There are 3 values possble to set, and they are all defualt on.
  • Can be Sold
    When checked, this will mark the Product as a sellable product on import.
  • Can be Purchased
    When checked, this will mark the product as a product than can be purchased.
  • Make to call
    When checked, this will mark the Procurement method as make to call on import.

    • Lifecycle Mappings

    In NTTY there are numerous stages and phases in a product lifecycle, however the PLM module in Odoo only caters for 5 stages. Therefore we have mapped each stage in NTTY to o certain stage in Odoo. These default mappings are provided though a XML-file and should be revised so it can be set to cater each organisation needs.
